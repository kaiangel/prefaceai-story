# PM 独立审查同事工作

作为 PM，对指定同事的工作成果进行毫无遗漏的独立深入审查。

**用法**：`/audit backend` 或 `/audit backend tester`

`$ARGUMENTS` 指定要审查的同事（一个或多个）。

## 执行步骤

### 1. 读取群聊

仔细阅读 `.team-brain/TEAM_CHAT.md`。

**群聊内容通常很长**，需要尽可能读取最新、尽可能多的行数（500-1000 行起），确保不遗漏任何与被审查同事相关的上下文。

### 2. 读取所有相关文档

读取所有在群聊中提到的、对应的以及必要的相关文档，包括但不限于：
- 被审查同事的 progress 文件（`.claude/agents/{agent}-progress/`）
- 被审查同事修改或创建的文件
- 相关的 team-brain 文档
- 任务派发时的需求描述和验收标准

### 3. 确保理解

确认自己完全理解了任务背景、需求、预期产出和验收标准。

**如果有任何困惑或不理解的地方，先提出来，不要带着疑问继续审查。**

### 4. 独立深入审查

毫无遗漏且全面、具体、清晰、细致地审查 `$ARGUMENTS` 指定的同事的工作：

- 任务是否完成？是否符合需求？
- 代码/文档质量如何？
- 有没有遗漏的边界情况？
- 有没有与其他同事的工作冲突或不一致？
- 验收标准是否满足？
- 有什么风险或隐患？

给出具体的审查结论和建议。
