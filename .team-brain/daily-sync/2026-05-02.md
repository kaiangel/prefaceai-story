### 3. 跨日 dev 服务死亡机制（永久教训）

macOS `pmset -g` standby=1 → 系统进入 standby → 所有用户进程被冻结/终止。**nohup 也躲不过 standby**。

修复方案:
- 短期: `nohup caffeinate -i -d -s &` 守护进程（防 idle sleep + display sleep + system sleep）
- 局限: 无法防合盖 / 重启 / OOM / 断电
- 长期: launchd plist / docker / 专用 dev 机（待评估）
