## 如何使用此文件

1. **初次迁移**: 花 30-60 分钟，把 Web 端 6 个对话的精华填充到上面
2. **重点提取**:
   - 重要的产品决策
   - 技术方案选择
   - 踩过的坑
   - 未来的想法
3. **持续更新**: 以后重要的讨论也记录在这里
4. **Agent 使用**: PM Agent 启动时读取这个文件，获得历史上下文

---

## 提取指南

### 值得记录的内容
- 为什么选择某个技术方案
- 产品方向的思考
- 用户需求的理解
- 重要的 trade-off 决策
- 曾经考虑但放弃的方案（为什么放弃）

### 不需要记录的内容
- 具体的代码实现细节（已在代码中）
- 临时的调试过程
- 已经失效的信息
