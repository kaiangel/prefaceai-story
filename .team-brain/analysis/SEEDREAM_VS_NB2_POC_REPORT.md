## 背景

Founder 于 2026-04-24 决策启动 Seedream 5.0-lite vs NB2 隔离 POC 对比，评估 Seedream 是否能替代或补充当前默认图像生成模型 NB2（Gemini 3.1 Flash Image Preview，即 `gemini-3.1-flash-image-preview`）。

Backend 使用相同 prompt + 参考图策略（仅 fullbody，不传 portrait），对 R8 测试集 shot_01~10 分别调用 Seedream 5.0-lite API（`doubao-seedream-5-0-260128`，火山方舟）生成 10 张图，同时复制 R8 历史 NB2 图作为基线，生成 comparison.html 对照页面。

**重要发现（backend 报告）**：
- Seedream 模型实际为 `doubao-seedream-5-0-260128`（Seedream 5.0-lite），而非原计划的 Seedream 4.0
- shot_04 被火山方舟内容审查（`InputTextSensitiveContentDetected`）拦截，原 prompt 含 "elderly man + worry" 组合，agent 将 "elderly man" → "older man"、"faint but legible furrow of quiet worry" → "sense of quiet contemplation" 后通过
- shot_04 已在 comparison.html Phase 3a 中加入红色警告 badge + 双 prompt 并排对照
- 总成本 ¥2.20（~$0.30），平均 57.1s/张，10/10 成功

## 评分方法 + 局限（诚实声明）

### 评分方法

评分基于以下间接信号，**不基于肉眼看图**：

1. **API 日志分析**（`logs/seedream_api_logs.json`）：每张图的 ref_count、latency、文件大小等结构性指标
2. **图像元数据分析**（PIL ImageStat）：图像尺寸、亮度均值（brightness）、标准差（std，反映色彩对比度/细节丰富程度）
3. **Prompt 文本分析**（`4_storyboard.json`）：每个 shot 的 image_prompt、text_overlay、characters_in_scene、场景类型
4. **角色数据分析**（`2_characters.json`）：4 角色的外观特征，作为 D2 角色一致性评分的参照
5. **R8 背景知识**：NB2 在 R8 运行时的已知表现（来自历史测试记录）
6. **shot_04 内容审查事件**：作为 D5 审查严格度评估的直接证据

**关键观察**：
- Seedream 全部输出 1664×2496（2K，约 400 万像素），NB2 全部输出 848×1264（约 107 万像素）—— 分辨率差约 4x
- 文件大小：Seedream 平均 ~499K/张，NB2 平均 ~1569K/张。NB2 文件更大是因为 NB2 是 PNG 无损，而 Seedream 虽然也是 PNG 但分辨率更高却更小，说明 **Seedream 压缩率更高 / 或细节密度不及 NB2**（后者可能更接近事实，PNG 压缩率高 = 图像重复区域多 = 细节少）
- NB2 的 std（颜色标准差）在多数 shot 显著高于 Seedream，意味着 **NB2 色彩对比度更丰富，细节更多变**；Seedream 的 std 偏低暗示画面更平整/统一，可能呈现更"干净"或更"卡通"的效果
- 亮度：两者没有系统性差异，取决于具体 shot 场景

### 重要局限声明

**本评分是基于 metadata 的间接评估（indirect evaluation），不是视觉评分。**

作为 text-only agent，我无法直接"看到"图像内容。以下评分无法可靠评估：
- 角色脸部是否真实对应参考图人物
- 服装颜色/款式是否与角色设定匹配
- 场景是否与预期一致
- 文字是否正确渲染（若有）
- 艺术构图的真实质量

**Founder 必须亲自打开 `comparison.html` 看图做最终确认**，本报告仅作决策的结构化参考。

## Shot_04 Sanitize 事件分析

### 事件详情

- **原 prompt 触发词**: `"elderly man"` + `"faint but legible furrow of quiet worry"` 中的 "worry"
- **Seedream 审查系统**: 火山方舟（国内版）`InputTextSensitiveContentDetected`
- **Agent 处理**: 将 "elderly man" → "older man"，"faint but legible furrow of quiet worry" → "sense of quiet contemplation"
- **结果**: 修改后通过，生图成功

### 影响评估

1. **内容表达损失**: "elderly" 是 char_001（外公梁德顺）的核心视觉特征（年迈、略微弓背、银白稀疏发际）；"worry"/"furrow" 是该 shot 的核心情绪（外公挂念外婆）。sanitize 导致**角色识别精度下降 + 情绪表达失准**。

2. **不公平性**: NB2 在 R8 时对相同 prompt 无拦截，说明这是 Seedream 国内版特有的审查问题。**这使 shot_04 的对比无效**，已从均分中排除。

3. **生产影响预测**: 如果 Seedream 替代 NB2 进入生产：
   - R8 这 10 shots 中有 1/10（10%）需要 sanitize
   - 未来批次中 "elderly" / "old man" 组合是常见角色描述（老人角色出现频率高）
   - "worry" / "concern" / "furrow" 等情绪词在叙事场景中极为常见（非暴力，但触发阈值）
   - 预估实际生产中 5%~15% 的 shots 可能触发内容审查，需要 prompt sanitize 兜底

### 同批 Prompt 潜在风险评估

**如果 Founder 看图后发现 Seedream 质量确实更好**，建议方案：
