## L1 工程并行化（M1，立即做）

### 现状（Code Forensics Agent 2026-04-25 审查）

```python
# pipeline_orchestrator.py:524-677 — Stage 5 主循环（当前实现）
for i, shot in enumerate(shots):          ← 纯串行 for
    result = await image_gen.generate_shot_image_phase2_safe(...)
    await asyncio.sleep(0.5)              ← 每张 0.5s 冷却
# 注释明文：# 控制并发（简单串行，可优化为并行）
```

### 死参数证据
- `pipeline_orchestrator.run(max_concurrent_images: int = 2)` — 签名定义了，**从未被读取**
- `config.py:32 IMAGE_MAX_CONCURRENT: int = 3` — **没接入图像生成主路径**
- `image_generator.py:1475 generate_batch()` — 已实现 `asyncio.Semaphore + asyncio.gather()`，**被孤立**

### 改造方案
1. **替换 Stage 5 主循环**：用 `image_generator.generate_batch(shots, max_concurrent=3)` 替换 for 循环
2. **Haiku validator 并行化**：每张图后的 `shot_validator.validate_shot()` 改异步并发
3. **保留 0.5s 冷却**逻辑（在 Semaphore 内部，避免突发）
4. **充分兜底**（用户强调）：429 / 限流 / 部分失败 / 全部失败的所有分支必须测试覆盖（详见 PENDING `TASK-PARALLEL-M1` 验收清单）

### 预期效果
- **耗时**：13.5 min → **~4.5 min**（达到 Midjourney Fast Mode 体验）
- **成本**：$1.85 不变（按张计价）
- **毛利率**：30% 不变
- **副作用**：UX 跃迁 → 付费意愿 / 留存 / 口碑全部改善 → 间接帮助毛利

### 风险
- **NB2 高峰期 429 失败率 ~30%**（Agent B 调研，2026-04 数据）
- 缓解：现有 `MAX_RETRIES=3` + `RETRY_DELAY=2s` + 限流退避翻倍 + Semaphore + asyncio.sleep(0.5)
- **必须 Backend 验证所有失败路径都有兜底**（验收时分项跑测）

### 工程量
1-2 天（Backend）

## L2 Credits 制混合订阅（M2-M3）

### 当前定价
- Pro ¥199/月 = 10 条配额
- 用户假设满额消耗 → 30% 毛利率（最差情况）

### 新定价
| 层 | 月费 | 含 Credits | 1 张 = 1 Credit | 超出价 |
|----|------|-----------|----------------|-------|
| Free | ¥0 | 60（3 条带水印） | — | 不可超 |
| Lite | ¥49 | 40（2 条 + 余额）| ¥1.5/Credit | ¥30/超出 |
| Pro | ¥199 | 200（10 条） | ¥1.2/Credit | ¥20/条 |
| Pro Max | ¥499 | 500（25 条）| ¥1.0/Credit | ¥18/条 |

### 关键设计
- **Credits 独立计算**：1 条故事 = 20 Credits（默认 20 张图）
- **未用 Credits 30 天结转**（防止用户感觉"亏"，提升续费）
- **超出按 Credit 计费**（边际成本 ¥0.5/张，超出价 ¥1-1.5/张，毛利 ~70%）

### 行业证据
- Runway / Midjourney / Pika 都是 Credits 制
- 用户实际消耗 = 配额 60-70%（不是 100%）
- 序话假设保守取 70%

### 数学
- Pro ¥199 实际消耗 7 条 × $1.85 = $12.95，毛利 = $14.4，**毛利率 53%**（vs 30%）
- 加上 30% 用户超出购买（按 ¥20/条）：毛利率综合到 **55-58%**

### 工程量
2-3 天（Backend：Credits 表 + 扣减逻辑 + 结转 + 超出购入口；Frontend：Stage A 显示余额、Stage E 结算）

### 与 L1 配合
- L1 让单条耗时 4.5 min，用户单次会话能完成更多创作 → Credits 消耗更快（增量定价空间）
- L2 让 Credits 制度合理化 → 不会因为"快"而引发"用太快用完了"的焦虑

## L3.a 产品减量（M3-M6）

### 思路
当前默认每条 20 张图。通过产品设计降低实际平均生成数：
- Stage D 默认先生成 **6 张样张** + Bullet 大纲
- 用户确认风格 / 故事走向后再生成剩余 14 张
- 假设 30% 用户在 6 张时就停（重启或放弃）→ 实际平均 14 张/条

### 数学
- 单条成本：$1.85 → **$1.30**（降 30%）
- 毛利率（Pro ¥199 实际消耗 7 条）：53% → **62%**

### 用户接受度
- 早期失败成本低（用户更愿意试）
- "先看 6 张再决定"符合人类决策心理（避免沉没成本）
- 需要小规模测试（10-20 内测用户）确认接受度

### 工程量
1 周（Frontend Stage D 重设计 + Backend 增量生成接口）

## L4.a NB2 Batch 部分场景（M9-M12，等 Google 修 Bug）

### 关键事实（Agent B 调研 2026-04-25）

| 维度 | 状态 |
|------|------|
| Google 官方文档 | 说支持 NB2 Batch，定价 $0.067 → **$0.034**（50% off）|
| 实际可用性 | **不可用** — Forum 多帖证实（2026-03/04 持续报告）：job 进入 `JOB_STATE_RUNNING` 永久挂起，`completionStats` 永远 `undefined`|
| 根因 | NB2 Dynamic Shared Quota 与 Batch 调度机制不兼容 |
| 修复时间表 | 无官方公告 |

### 替代方案（已被 DEC-022 否决）
- gemini-2.5-flash-image Batch 可用，$0.0195/张（降 71%）
- 但画质明显低于 NB2 → 序话护城河不能动

### 等 Bug 修复后的部署计划
- **C 端实时主路径不动**（保护核心 UX 体验）
- **B 端 / API**：可选 batch 模式（24h 等待，享 30% 折扣）
- **章节预生成**（用户保存草稿后台生）：100% batch
- **风格库扩展预览**（80+ 风格的样图）：100% batch

### 数学（如 Bug 修了）
- 部分流量走 batch（假设 40% 流量）
- 综合单条成本：$1.10 × 0.6 + ($1.10 × 0.5) × 0.4 = $0.88
- 毛利率：70% → **76%**

### 监控
- **每月一次**复测 Google AI Developers Forum + 自测 NB2 Batch 是否恢复
- 跟进 Issue 链接：
  - https://discuss.ai.google.dev/t/batch-api-stalls-indefinitely-with-gemini-3-1-flash-image-preview/136095
  - https://discuss.ai.google.dev/t/gemini-batch-jobs-getting-stuck-for-flash-3-and-3-1-pro/130602

## L4.b 自建 SD 集群（Pre-A 后，M18+）

### 思路
- H100 / A100 集群（自建 or RunPod / Lambda Labs 租）
- 模型：Stable Diffusion 3 Large + 角色 LoRA（自训）
- 单图成本：$0.067 → **$0.005-0.015**（降 80-90%）
- 单条总成本：$1.85 → **$0.30-0.50**

### 前置条件
- **必须先做 PoC 验证**：Stable Diffusion 3 + 自训角色 LoRA 的多角色一致性能否到 NB2 水平 85%+（NB2 是 95-100%，目标 85% 在 60% 成本下可接受）
- 投入：$5-10K（PoC 阶段，M9-M12 启动验证）
- 后续投入：$50-100K（Pre-A 后规模部署）

### 时机
- 月生图量 ≥ **50 万张**才划算（自建集群固定成本）
- 这对应 Q4 末 / 2027 Q1 用户规模
- Pre-A 后才大规模投入，天使轮内只做 PoC 验证

### 数学
- 单条成本：$1.10 × 0.5 + $0.40 × 0.5 = $0.75（混合 50% 自建 50% NB2）
- 进一步：100% 自建 → $0.40
- 毛利率：76% → **85%**（接近 SaaS 行业基准）

### 团队
- 需要专职 ML 工程师（M9 启动 PoC 时由 CTO Ben 亲自带 + 1 名 ML 候选人 PoC 时介入）
- Pre-A 后正式 hiring
