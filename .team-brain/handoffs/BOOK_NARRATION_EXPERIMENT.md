## 背景

我们想验证：**序话Story能否用于生成书籍解说视频**。

书籍解说视频（抖音/B站常见形态）：
- 把一本书的精华用3-5分钟视频讲完
- 旁白驱动 + 配图辅助
- 目标是"让没读过这本书的人快速了解核心内容"

这和短剧的核心差异：

| 维度 | 短剧 | 书籍解说 |
|------|------|---------|
| 驱动 | 剧情冲突、角色对话 | 信息传递、知识点串联 |
| 角色 | 核心（需要高一致性） | 配角或无角色 |
| 画面 | 电影感场景 | 概念可视化、信息图 |
| 节奏 | 情绪起伏 | 娓娓道来 |

---

## 目标

**最小改动验证可行性**：不动现有代码，用新Prompt跑通一个书籍解说demo。

## 技术方案：Prompt层面适配

### 核心洞察

书籍解说和短剧的**服务架构完全可以复用**，差异只在**Prompt层面**：

```
短剧流程：
idea → 故事大纲 → 角色设计 → 分场剧本 → 分镜脚本 → 图片

书籍解说流程：
书籍内容 → 要点提炼 → (可选)人物提取 → 解说脚本 → 配图分镜 → 图片
          ↓           ↓                ↓           ↓
       复用Stage1   可跳过Stage2    复用Stage3   复用Stage4/5
```

### 方案：新建书籍解说Prompt模板

在 `app/prompts/` 下新建 `book/` 目录：

```
app/prompts/book/
├── book_outline_prompt.py      # 书籍要点提炼（替代story_outline）
├── book_narration_prompt.py    # 解说脚本（替代screenplay）
└── book_storyboard_prompt.py   # 配图分镜（替代storyboard）
```

### 各Stage的Prompt改造要点

#### Stage 1: 书籍要点提炼

**输入**：书籍标题、作者、简介、核心内容（用户提供）

**输出**：
```json
{
  "book_title": "人类简史",
  "author": "尤瓦尔·赫拉利",
  "core_theme": "人类如何从无名小卒成为地球主宰",
  "key_insights": [
    {
      "insight_id": 1,
      "title": "认知革命",
      "summary": "7万年前，智人突然开始创造虚构故事...",
      "visual_concept": "一群原始人围着篝火讲故事，篝火的烟雾中浮现神话生物"
    },
    {
      "insight_id": 2,
      "title": "农业革命是史上最大骗局",
      "summary": "小麦驯化了人类而非人类驯化了小麦...",
      "visual_concept": "一个农民弯腰种地，对比之下猎人轻松行走"
    }
  ],
  "target_audience": "对历史和人类发展感兴趣的年轻人",
  "tone": "启发性、略带幽默的知识科普"
}
```

**Prompt关键指令**：
- 提炼3-5个核心洞见（不是情节点）
- 每个洞见必须有`visual_concept`字段（便于后续配图）
- 保持原书的核心观点，不要过度简化

#### Stage 2: 跳过或简化

书籍解说大多数情况**不需要角色设计**。

但如果书中有重要人物（如传记类），可以简化处理：
- 只提取书中提到的人物
- 不需要完整的physical/clothing描述
- 用通用描述即可（如"一个穿着西装的中年男性企业家"）

**建议**：先跳过，看效果再决定是否补充。

#### Stage 3: 解说脚本

**输入**：Stage 1的要点提炼结果

**输出**：
```json
{
  "narration_segments": [
    {
      "segment_id": 1,
      "insight_ref": 1,
      "narration_text": "七万年前的某一天，一个智人对他的同伴说了一句话，改变了整个人类的命运。他说的不是"小心老虎"，而是"我们部落的守护神会保佑我们"。这是人类第一次讲述虚构的故事...",
      "duration_hint": 25,
      "visual_direction": "开场：原始人围坐篝火，镜头缓缓推进到火焰，火焰中浮现模糊的神话形象"
    }
  ],
  "total_duration": 180,
  "opening_hook": "如果有人告诉你，小麦才是地球上最成功的物种，你会相信吗？",
  "closing_statement": "这就是人类简史告诉我们的：我们统治世界，不是因为我们最强壮，而是因为我们最会讲故事。"
}
```

**Prompt关键指令**：
- 用口语化的叙述风格（适合TTS朗读）
- 每段有明确的`visual_direction`（给Stage 4用）
- 开头要有hook，结尾要有总结
- 控制每段时长（建议20-40秒/段）

#### Stage 4: 配图分镜

基本复用现有StoryboardDirector，但Prompt调整：

**差异点**：
- 不强调"角色一致性"，强调"概念可视化"
- 镜头语言偏静态（知识传递不需要动作戏）
- 可能需要更多的"意象化"表达

**新增指令**：
```
CONCEPT VISUALIZATION RULES:
- 抽象概念必须转化为具体场景
- 例如："认知革命" → 原始人围坐篝火，烟雾中浮现虚构形象
- 例如："农业陷阱" → 对比画面：弯腰种地的农民 vs 轻松狩猎的猎人
- 避免纯文字/图表，用故事化场景表达
```

#### Stage 5: 图片生成

**完全复用现有能力**，但注意：
- 风格选择：建议用`illustration`或`digital_art`，比`realistic`更适合概念表达
- 可以跳过角色参考图生成（如果没有固定角色）
- 场景参考图仍然有用（保持视觉统一）
