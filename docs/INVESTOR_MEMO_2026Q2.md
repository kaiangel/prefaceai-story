## 一、一句话定义

**序话 Story** 是一个 AI 驱动的图文/短视频一键生成平台：用户输入一句话创意，系统自动完成编剧、角色设计、分镜绘制、配音配乐，输出可直接发布的条漫或短视频。

品牌 Pipeline：**FrameSpark™**

## 五、使用场景（优先级排序）

### 优先级 1：个人内容创作者
- 与 683 内测用户画像高度吻合（个人创作者，无专业技能，有故事创意）
- 典型场景：一句话 idea → 20 分钟内产出一条可发布的条漫/短视频
- 核心痛点：会想故事，不会画图、不会剪视频、不会配乐
- 变现路径：订阅制（月卡 / 年卡）

### 优先级 2：MCN / IP 工作室
- Amy（FanBook）有直接资源：中小 MCN、创作者工作室
- 典型场景：批量生产系列故事内容，维持更新频率
- 差异化：FrameSpark 支持角色一致性跨集保持（100% 3 人场景），适合系列 IP
- 变现路径：团队版订阅 + API 调用量计费

### 优先级 3：教育（K12 故事绘本 / 英语启蒙）
- 绘本生成：图文结合，儿童风格支持完善（80+ 风格含儿童绘本风格）
- 英语启蒙：对话条漫 + 朗读 TTS，语音对齐 ≤80ms 即可用于教学产品
- 变现路径：B2B SaaS（教育机构）或 B2C 订阅

### 延展场景（中长期）
- 出版社 / 漫画公司：批量数字内容生产
- 品牌方：故事化营销内容（条漫广告）
- 短剧工作室：剧本可视化 / 分镜预生产

## 八、技术护城河摘要

| 护城河 | 描述 | 竞争对手复制难度 |
|--------|------|:--------------:|
| **FrameSpark 5 阶段 Pipeline** | Story→Character→Screenplay→Storyboard→Image 端到端闭环，单点可换模型，整体不可轻易复制 | 高（需系统化 Prompt 工程 + 架构经验） |
| **100% 3 人角色一致性** | 行业首个工程化落地多角色一致性（NB2 + 串行参考图生成）| 中高（NB2 公开，但工程化需要深度调优） |
| **音画对齐 ≤80ms** | Whisper word-level 时间戳 + 自研多策略对齐 | 中（技术可复制，但需积累调优经验） |
| **80+ 风格预设** | StyleEnforcer mandatory/forbidden 词系统，跨风格通用 | 中（词库可枚举，但质量调优需大量测试） |
| **Prompt Engineering 积累** | Kai 的 Prompt Engineering 行业领先经验固化为 Pipeline 规则 | 高（非公开，个人认知 + 迭代历史） |
